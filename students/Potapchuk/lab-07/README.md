## Цель работы

Реализовать паттерн **CQRS** (Command Query Responsibility Segregation) с чётким разделением моделей записи и чтения для системы «Электронная библиотека».

---

## Вариант №6 — Электронная библиотека

**Ядро домена:**
- Агрегат: `UserLibrary`
- Сущность: `Book`
- Value Objects: `ReadingProgress`, `Rating`

---

## Ход выполнения работы

### 1. Write Model (Командная модель)

Write Model предназначена для обработки команд, реализации бизнес-логики и поддержания инвариантов системы. В ней происходит генерация доменных событий.

### 2. Read Model (Модель чтения)

Read Model представляет собой денормализованную структуру данных, оптимизированную исключительно для быстрых запросов и чтения информации пользователем.

### 3. Event-Driven Synchronization (Проекции)

Синхронизация между Write Model и Read Model осуществляется через обработку доменных событий. При возникновении событий в Write Model специальные проекции обновляют Read Model.

### 4. Оптимизация запросов

Для ускорения чтения используется денормализация данных и материализованные представления, что позволяет избежать сложных соединений таблиц при каждом запросе.

---

## Пирамида CQRS

| Модель         | Назначение                          |
|----------------|-------------------------------------|
| Write Model    | Бизнес-логика, инварианты, команды |
| Read Model     | Быстрые запросы и отображение данных |
| Projection     | Синхронизация моделей через события |

---

## Контрольные вопросы

**1. CQRS vs CQS**  
CQS — это принцип разделения методов (метод либо изменяет состояние, либо возвращает данные).  
CQRS — архитектурный подход, при котором модели записи и чтения разделены на уровне архитектуры.

**2. Почему Read Model денормализована?**  
Денормализация позволяет значительно ускорить операции чтения и уменьшить количество соединений таблиц (JOIN).

**3. Как синхронизировать модели при сбое?**  
Синхронизация осуществляется через повторную обработку событий, использование очередей и идемпотентные обработчики проекций.

**4. Eventual Consistency**  
Модель чтения может временно отличаться от модели записи, однако со временем данные обязательно приходят в согласованное состояние.
