## Цель работы

Реализовать инфраструктурный слой (Infrastructure Layer) приложения Request Service с использованием:
- Репозитория на базе Entity Framework Core + PostgreSQL
- REST API контроллера (ASP.NET Core)
- Публикации доменных событий
- Docker Compose для запуска базы данных

---

## Ход выполнения работы

### 1. Настройка базы данных и ORM

- Создан `AppDbContext` (Entity Framework Core)
- Реализована конфигурация сущности `Request` (`RequestConfiguration`)
- Настроено подключение к PostgreSQL через `appsettings.json`

### 2. Repository (Исходящий адаптер)

Реализован класс `RequestRepository`, который имплементирует интерфейс `IRequestRepository` из Application Layer.

**Основные методы:**
- `SaveAsync()` — сохранение/обновление агрегата
- `GetByIdAsync()` — поиск по ID
- `GetAllActiveAsync()` — получение активных заявок

### 3. REST Controller (Входящий адаптер)

Создан `RequestController` с необходимыми эндпоинтами:

| Метод | Эндпоинт                        | Описание                    |
|-------|---------------------------------|-----------------------------|
| POST  | `/api/requests`                 | Создать заявку              |
| POST  | `/api/requests/{id}/assign-group` | Назначить группу         |
| POST  | `/api/requests/{id}/activate`   | Активировать заявку         |
| POST  | `/api/requests/{id}/complete`   | Завершить заявку            |
| GET   | `/api/requests/{id}`            | Получить заявку по ID       |
| GET   | `/api/requests`                 | Список активных заявок      |

### 4. Event Publisher

Реализован `InMemoryEventPublisher` для публикации доменных событий в консоль (может быть заменён на RabbitMQ/Kafka).

### 5. Docker Compose

Создан `docker-compose.yml` для быстрого запуска PostgreSQL.

### 6. Регистрация зависимостей

Все компоненты инфраструктурного слоя зарегистрированы в `Program.cs` через Dependency Injection.

## Контрольные вопросы

**1. Почему Repository находится в Infrastructure, а не в Domain?**  
Repository — это техническая деталь хранения данных. Domain не должен знать о базе данных, ORM или SQL.

**2. В чём преимущество ORM над обычным SQL?**  
Автоматическое маппирование объектов, работа с отношениями, миграции, защита от SQL-инъекций, кросс-платформенность.

**3. Зачем использовать миграции вместо ручного CREATE TABLE?**  
Миграции позволяют версионировать структуру БД, безопасно обновлять её в команде и на продакшене.

**4. Как testcontainers упрощает интеграционное тестирование?**  
Позволяет запускать реальную БД (Docker) внутри тестов без установки и загрязнения основной базы.

**5. Почему REST Controller не вызывает напрямую доменные модели?**  
Controller должен только принимать HTTP-запрос, делегировать работу Application Service и возвращать ответ. Это сохраняет чистую архитектуру и разделение ответственности.
