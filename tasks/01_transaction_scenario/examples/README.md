## Пример адаптации

**Если ваш вариант:** "Микро-шоп «Добавь в корзину» 🛒"

**Было (в примере):**
```gherkin
Scenario: Успешное создание заявки и формирование группы
  Given пользователь авторизован как "petrov@rescue-southwest.by" с ролью "Координатор"
  And в системе есть 5 волонтёров со статусом "AVAILABLE"
  When координатор создаёт заявку "Иван Иванович Петров"
  And формирует группу из 4 участников
```

**Стало (ваша адаптация):**
```gherkin
Scenario: Успешное оформление заказа
  Given пользователь авторизован как "buyer@example.com"
  And в корзине есть товар "Ноутбук ASUS" количеством 1
  When покупатель нажимает "Оформить заказ"
  And вводит адрес доставки "Минск, пр-т Независимости, 1"
```

## Уровень детализации

**Минимальный уровень (для зачёта):**
- Use-case: основной поток + 2 альтернативных + 2 исключительных
- Gherkin: 4 сценария (1 happy + 3 error)
- Analysis: таблица транзакций + 2 описания обработки ошибок

**Продвинутый уровень (для высокой оценки):**
- Use-case: все возможные потоки, открытые вопросы, частота использования
- Gherkin: 10+ сценариев, граничные случаи, идемпотентность
- Analysis: диаграммы потоков, метрики, риски, компромиссы, несколько транзакций

## Диаграммы последовательности (Sequence Diagrams)

⚠️ **Важно:** Примеры диаграмм в формате PlantUML находятся в папке [diagrams/](diagrams/)

👉 **Смотрите:** [diagrams/README.md](diagrams/README.md) - инструкции по экспорту в PNG

**Готовые диаграммы:**
- [sequence-happy.puml](diagrams/sequence-happy.puml) - Успешный сценарий (2 транзакции)
- [sequence-error-notification.puml](diagrams/sequence-error-notification.puml) - Сценарий с ошибкой SMS

**Где взять инструменты:**
- **PlantUML онлайн:** https://www.plantuml.com/plantuml/uml/
- **VS Code плагин:** PlantUML (jebbs.plantuml) - автопредпросмотр + экспорт
- **Экспорт в PNG:** `Alt+D` для preview → `Ctrl+Shift+P` → "PlantUML: Export Current Diagram" → PNG

**Пример минимальной диаграммы (для Search & Rescue домена):**

Смотрите файлы в [diagrams/](diagrams/) - там полные примеры для успешного сценария и error case с комментариями на русском языке.

**Основные компоненты в диаграммах:**
- **Координатор** - инициирует процесс
- **Request Service** - создаёт заявку (Tx1)
- **Geo Planning Service** - разбивает на зоны
- **Group Service** - формирует группу (Tx2)
- **PostgreSQL** - две отдельные транзакции
- **Event Bus** - асинхронная публикация событий
- **Notification Worker** - обработка уведомлений с retry

**Пример с несколькими транзакциями:**

Смотрите [diagrams/sequence-happy.puml](diagrams/sequence-happy.puml) - там показано:
- **BEGIN/COMMIT** для каждой транзакции
- **Pessimistic locking** (SELECT FOR UPDATE)
- **Асинхронная обработка** через Event Bus
- **Retry логика** для SMS-уведомлений

## Частые ошибки

### ❌ Ошибка 1: Слишком поверхностное описание
**Плохо:**
```
Операция: Создание заявки
Статус: Синхронная
```

**Хорошо (как в примере):**
```
| Операция | Синхронная/Асинхронная | Откат при ошибке | Retry-стратегия | Идемпотентность |
| Создание записи Request в БД | Синхронная | Да (ROLLBACK транзакции №1) | Нет (контролируется БД) | Да (проверка дублей по idempotency_key) |
```

### ❌ Ошибка 2: Игнорирование ошибок
Описали только happy path, забыли про ошибки - это **не пройдёт защиту**!

**Хорошо:** В примере есть обработка сбоев БД, Notification Service, Geo Planning Service, конкурентных обновлений волонтёров.

### ❌ Ошибка 3: Копирование примера без адаптации
Если в вашем отчёте фигурируют "заявки", "волонтёры" и "зоны", хотя ваш вариант - "Бронирование отелей" - **0 баллов**.

### ❌ Ошибка 4: Одна большая транзакция
Если вы запихнули всё (создание заявки + группы + уведомления) в одну транзакцию - это неправильно!

**Хорошо:** В примере две транзакции: Tx1 (заявка + зоны), Tx2 (группа + назначение). Уведомления - асинхронно.

## Контрольный чек-лист

Перед сдачей проверьте:

- [ ] Use-case содержит акторов, предусловия, постусловия
- [ ] Описаны минимум 2 альтернативных потока
- [ ] Описаны минимум 3 исключительных ситуации
- [ ] Gherkin-сценарии покрывают happy path и ошибки
- [ ] Таблица транзакционных границ заполнена (мин. 8 операций)
- [ ] Описана обработка минимум 3 исключительных ситуаций (с компенсациями)
- [ ] Если есть несколько агрегатов - разделены на отдельные транзакции
- [ ] Все термины соответствуют **вашей** предметной области
- [ ] PlantUML диаграммы экспортированы в PNG
- [ ] Файлы названы правильно (не "пример.md", а "use-case.md")
